# Multimodal Clinical Query Router

> Route natural language oncology questions to the right retrieval modality —
> semantic search, structured lab queries, NGS mutation lookup, or cross-modal joins —
> then generate grounded answers with LLM-as-judge faithfulness scoring.

---

## Motivation

Clinical informatics platforms (Tempus, Foundation Medicine, Flatiron) store
oncology data across fundamentally different modalities:

| Modality | Store | Content |
|---|---|---|
| **TEXT** | Vector DB (ChromaDB) | Physician progress notes, assessments |
| **LABS** | Structured table (Pandas) | CBC, CMP, tumor markers — 8-visit time-series |
| **GENOMIC** | Profile store (JSON) | FoundationOne CDx NGS reports, TMB, MSI |
| **FUSED** | Cross-modal join | Patient-ID join across all three sources |

A naive RAG pipeline routes everything to the vector store, polluting results
with unstructured note text when a clinician asks *"which patients have CRP above 10?"*
This project implements a lightweight, rule-based router that classifies each
incoming query into the correct modality before retrieval — no fine-tuning required.

## Architecture

```
Query
  │
  ▼
┌─────────────────────────────────┐
│  Router (router.py)             │  Regex signal detection
│  TEXT / LABS / GENOMIC / FUSED  │  → modality + extracted filters
└─────────────────────────────────┘
  │
  ▼
┌──────────────────────────────────────────────────────────────┐
│  Retriever (retrievers.py)                                   │
│                                                              │
│  TEXT    → ChromaDB cosine similarity (DefaultEmbedding)     │
│  LABS    → Pandas .eval() filter + time-series aggregation   │
│  GENOMIC → Exact/fuzzy gene/variant match on NGS profiles    │
│  FUSED   → Cross-modal patient-ID join (all three sources)   │
└──────────────────────────────────────────────────────────────┘
  │
  ▼
┌─────────────────────────────────────────────────────────────┐
│  Answer Generator (answer_generator.py)                     │
│  Groq llama-3.3-70b-versatile                               │
│  Grounded clinical answer + LLM-as-judge faithfulness eval  │
└─────────────────────────────────────────────────────────────┘
```

**Faithfulness metrics** (LLM-as-judge, per query):
- `grounding_score` — fraction of answer facts traceable to retrieved evidence
- `hallucination_risk` — inverse; flags fabricated patient data
- `completeness` — coverage of all question aspects

## Dataset (synthetic, HIPAA-safe)

Generated by `scripts/generate_data.py`:

- **25 patients** across 5 cancer types (NSCLC, CRC, Breast, PDAC, GBM)
- **200 lab records** — 8 visits per patient; CBC, CMP, CEA, CRP, LDH
- **25 genomic profiles** — FoundationOne CDx format; TMB, MSI, PD-L1 TPS
- **25 clinical notes** — structured progress notes with ECOG, staging, treatment, AEs

All data is entirely synthetic. No PHI. No real patients.

## Quickstart

```bash
# 1. Clone
git clone https://github.com/Rabba-Meghana/Multimodal-Clinical-Query-Router.git
cd Multimodal-Clinical-Query-Router

# 2. Install
pip install -r requirements.txt

# 3. Get a free Groq key (takes 30 seconds): https://console.groq.com
export GROQ_API_KEY=your_key_here

# 4. Generate synthetic data
python scripts/generate_data.py

# 5. Run a query
python run.py --query "Which EGFR patients have CRP above 10 and documented progression?"

# 6. Launch the Streamlit UI
streamlit run app.py
```

## Why This Matters for Clinical AI

Most production oncology RAG systems fail on structured queries because they treat
all data as text. This router demonstrates:

1. **Query intent classification** without LLM overhead (rule-based, <1ms)
2. **Heterogeneous store retrieval** — cosine similarity is wrong for lab thresholds
3. **Cross-modal join** — linking a patient's genomic profile to their lab trajectory
4. **Faithfulness evaluation** — critical for clinical decision support safety
5. **Grounded generation** — system prompt enforces citation of patient IDs

These are the exact challenges faced by platforms like Tempus AI, Flatiron Health,
Foundation Medicine, and PathAI when deploying LLMs on real oncology data.
